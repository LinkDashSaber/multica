# 生命周期 opt-in：只有指派给 Workflow 的 issue 进入 Raven 轨道

issue 指派选择器扩展为三类：成员 / 智能体 / Workflow。决定：九状态生命周期、门禁、证据链、审查包只覆盖**指派给 Workflow** 的 issue；直接指派给裸智能体或小队的 issue 保持 multica 原生行为，Raven 不介入。理由：不破坏现有体验、团队可渐进采纳、避免改 multica 核心派单路径（合并冲突面最小）。曾考虑 all-in（所有 issue 强制走生命周期），因破坏轻量任务体验且侵入核心代码而放弃；按项目配置强制开关留作 v2 演进。

配套的升轨机制：裸智能体/小队跑完一次需求后，平台提议"沉淀为 workflow"等用户确认——自由轨道到治理轨道的迁移靠吸引而非强制。
