# 10. 从零创建 workflow 走内置 authoring 策略，不做表单/代码编辑入口

日期：2026-07-05

## 状态

已接受

## 背景

ADR-0007 排除了拖拽编辑器；沉淀是 workflow 的主要来源，但把现成 SOP 搬进
平台需要一个从零创建入口。表单 + 代码框把合同思维的负担压给用户；UI 里的
"贴代码"逃生舱与仓库 PR 路径重复。

## 决定

"新建交付策略" = 创建一条指派给内置 workflow-authoring 策略的需求：澄清
节点追问阶段/拍板点/预算 → agent 起草 defineWorkflow 文件 → spec-confirm
拍板点确认合同摘要 → PR → 合并即注册。用户只回答问题和拍板，不写代码。
工程师手写走仓库 PR，UI 不重复提供。

注册动作挂在 authoring 需求的 Merged 生命周期事件上（控制面已追踪该需求），
不依赖仓库 webhook——需求闭环驱动注册。

## 后果

- 创建体验 = 使用体验：用户首次创建即在见识 workflow 的运行方式。
- 创建与沉淀（升轨/改进 PR）在中途汇合为同一条管道（authoring 需求 →
  PR → Merged 注册），只维护一套。
- 沉淀出的 clarification 策略是这条链前半段的直接复用。
- 依赖：拍板点统一（澄清问答须是一等拍板点）；Merged 事件注册钩子。
