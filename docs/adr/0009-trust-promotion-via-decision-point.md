# 9. 信任额度经拍板点申请晋升，不自动降级

日期：2026-07-05

## 状态

已接受

## 背景

术语表定义了信任额度（workflow × 项目维度累积的自治等级）与生产线（门禁已
降级的 workflow），但从未实现。门禁降级是安全边界的变更：全自动降级对企业
场景太莽，纯手动开关又没人记得去开。

## 决定

workflow 满足晋升条件（同一门禁连续 8 次人审零驳回）时，平台经拍板点机制
向人发出晋升申请信，附完整审查记录作为证据；人批准后该门禁降级为抽查
（固定 1/5，不开放配置）；抽查失手一次立即自动升回全审。降级状态随时可
手动收回。

## 后果

- 安全边界的每次变更都有人签字、有据可查、可回退。
- 平台治理决策复用自己的拍板点原语（自指），无需新的审批机制。
- 列表页统计升级为晋升进度呈现（"距生产线还差 N 次零驳回"）。
- 需要按 workflow × gate 维度累积连续通过计数，驳回即清零。
